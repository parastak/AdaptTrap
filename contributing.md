# Contributing to AdaptTrap

Thank you for your interest in contributing to AdaptTrap!
We welcome contributions from researchers, engineers, students, and cybersecurity enthusiasts.

---

## Ways to Contribute

- Report bugs
- Improve documentation
- Add new attacker models
- Add new deception strategies
- Improve RL algorithms
- Optimize performance
- Add visualizations
- Improve tests

---

## Development Setup

```bash
git clone https://github.com/parastak/AdaptTrap.git

cd AdaptTrap

pip install -r requirements.txt
```

Run the sanity tests before submitting changes.

```bash
python sanity_test.py
```

---

## Pull Request Guidelines

Before opening a PR:

- Keep changes focused and small.
- Follow existing code style.
- Add comments where appropriate.
- Update documentation if needed.
- Ensure tests pass.

---

## Reporting Bugs

When opening an issue, please include:

- Operating System
- Python Version
- Error Message
- Steps to Reproduce
- Expected Behavior

---

## Feature Requests

Feature requests are welcome.

Please explain:

- What problem it solves
- Why it would benefit the project
- Possible implementation ideas

---

## Code Style

- Follow PEP 8
- Use meaningful variable names
- Keep functions small and readable
- Write docstrings where appropriate

---

## Questions

If you have questions, feel free to open a GitHub Discussion or Issue.

Thank you for contributing ❤️
